## 功能页面

1. **人员分析页**（默认）：总员工数、站点数、职位类型、待处理申请数概览；站点人员分布柱状图、职位分布柱状图
2. **员工列表页**：按线路 + 站点、职位、关键词搜索筛选员工，展示工号/姓名/性别/年龄/职位/线路/站点/部门/学历/入职日期/工龄/状态
3. **换站申请页**：申请列表展示（原线路站→目标线路站），支持状态筛选；提交换站申请（填员工编号、选目标线路+站、理由）；审批（通过/驳回）
4. **站点管理页**：站点列表（线路+站名+员工数），点击查看详情展示该站点下所有员工

## 后端对接

- API 基址由 FULLSTACK_API 块自动管理（`API_BASE`）
- 全部请求用 `apiGet`/`apiPost`/`apiPut`，路径不带 `/api` 前缀
- 员工/站点数据来自 `basic_data.metro_employees`（MySQL）
- 换站申请读写 `allocation.station_transfers`（MySQL）
